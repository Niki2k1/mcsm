The **Configuration** tab edits everything about how a server presents itself
and plays — and saving recreates the container with the new settings while
keeping the world.

## Write a rich MOTD

Try it — this is the same format MCSM's editor produces, rendered live in the
Minecraft font:

:demo-motd-editor

The MOTD editor in MCSM is a rich-text editor for Minecraft's server list
message:

- **Colors and formatting** — pick from the classic 16 `§` colors, bold,
  italic, underline, strikethrough and obfuscated text.
- **Hex colors** — full 24-bit color for Minecraft 1.16+ via `§x` sequences.
- **Live preview** — see the MOTD rendered in the Minecraft font as you type,
  including the animated obfuscated-text effect.
- **Two lines** — just like the real server list.

Codes are saved as standard `§`-sequences, so the result works with any
server type.

## Configure the world and game rules

Everything `server.properties` covers, in a form:

- World name, seed, world type and spawn protection radius
- Difficulty, game mode, max players and hardcore mode
- PvP, online mode, command blocks, flight
- Spawning toggles for animals, monsters and villagers
- Structure generation and the Nether

Minecraft versions are pulled live from
[`minecraft-data`](https://github.com/PrismarineJS/minecraft-data), so new
releases and snapshots show up automatically.

## Set custom environment variables

The **Environment** tab exposes the underlying
[itzg/minecraft-server](https://docker-minecraft-server.readthedocs.io/)
container directly:

- Add any environment variable the image supports — there are hundreds of
  options beyond what the UI models.
- See the full list of variables MCSM already sets, so you know what's safe
  to override.

::warning
Custom environment variables are applied on the next configuration save,
which recreates the container.
::
